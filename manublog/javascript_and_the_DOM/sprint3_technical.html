Analogy to describe the differences between HTML and CSS

Once upon a time, there were two friends. Their names were HTML and CSS. As a job, they were both in construction, and were building a house together.
On the house-building project, HTML and CSS both had different skills and responsibilities.
HTML was in charge of building the structure of the house - he built all the walls, the floor, the roof, and the doors and windows.
CSS was in charge of all the painting and decorating of the house - he chose the carpets, the wallpaper, the colour and shape of the window- and door-frames, and the design of the roof.

This house was also for themselves! They lived in the finished house happily ever after!

Control Flow and Loops

Control Flow is the term we use to describe the order in which the computer executes the statements.
It's important because if we want something to happen we can control it by using the different statements, for example: if, else, else if.
If we want to repeat the statments we have to use the loops, so we can use them over and over again also using different statements, for example: for, while.

Describe what the DOM is and an example of how you might interact with it

The DOM (Document Object Model) is a independent interface that treats an HTML and CSS document as a tree structure where each node is an object representing a part of the document.
These objects can be manipulated, that means we can access, create, remove and change them on the DOM via Javascript.
We can also interact with our code via Devtools, by using this tool we can create, remove and change statements in any website code, but after we refresh the site the elements will return to the original state.

Difference between accessing data from Arrays and Objects

To access data from an Array, you refer to it with the index number.
To access data from an Object, you have to refer to the object by using a dot or brackets.

what are Functions and why are they useful?

Functions can be described as a "formula" where we can organize our code to do a specific action and then reuse it as many times as we want.
